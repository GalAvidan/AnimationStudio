# Explainer Starter Spec

## Project

- Project name: `explainer-starter`
- Source script: `scripts/explainer-starter_script.md`
- Target runtime: 10 seconds
- Audience: creators who want explanations to be easier to understand
- Core message: animation is useful when it makes an idea clearer, not just prettier

## Visual Philosophy

Use simple geometric shapes, clean type, and calm motion. The animation should feel like an idea becoming organized. Avoid decorative complexity.

## Beat Map

| Beat | Time | What is said or implied | What the viewer sees | Purpose |
|---|---:|---|---|---|
| 1 | 0:00-0:02 | An idea can arrive all at once | Several small cards appear in a loose cluster | Show the initial confusion |
| 2 | 0:02-0:04 | Animation slows the idea down | The cards spread into a readable sequence | Show pacing and control |
| 3 | 0:04-0:07 | Problem, structure, connection | Three labeled steps appear one by one | Make the process explicit |
| 4 | 0:07-0:10 | The goal is clarity | The steps connect into one clean line with the final message | Land the takeaway |

## Key Moments

- The messy cluster becoming ordered is the main visual transformation.
- The final message should hold long enough to read.

## Audio Or Narration Sync Points

| Sync point | Time | Visual event |
|---|---:|---|
| messy-idea | 0:00 | Cards appear in a cluster |
| slow-down | 0:02 | Cards begin to spread out |
| three-steps | 0:04 | Step labels appear |
| final-message | 0:07 | Final line begins to appear |

## Assets

- Existing assets: none
- Assets to create: simple cards, step labels, connecting line
- Fonts: system sans-serif
- Colors: white background, dark text, blue accent, green success accent

## Constraints

- Runtime: 10 seconds
- Aspect ratio: 16:9
- Resolution: 1920x1080
- Frame rate: 30fps
- Must include: final line, "Clearer, not just prettier."
- Must avoid: decorative effects that distract from the explanation

## Review Checklist

- The core message is clear.
- Every beat supports the explanation.
- Text is readable.
- Motion guides attention.
- No implementation details are included too early.
- The build can be derived from this spec.
